Basic grid

Columns must be wrapped inside a .row element and must be at least assigned a .column class. By default the grid element will divide the width equally amongst its child columns.

The columns will be equal in height by default.

Grid gutter

By default there are no gutters between columns. By adding .gutter to your .row element, all child columns will receive a gutter in between.

Container

To complement your basic grid system, you can create a .container wrapper. By default the grid expands to the full width of its parent. To provide a left-right responsive margin, nest the grid inside a .container element.

Don't add a .container class to the row element. Keep it separate. This will conflict with the way the container deals with its margins. 

<div class="container">
    <!-- Insert rows and columns here -->
</div>

        <div class="column-5">5</div>
        <div class="column-4">4</div>

    </div>
</div>

Spacing

We define two types of spacings: the spacing of elements inside their parent container (margins) and spacing inside the element (paddings).

.stack

Handle vertical margins with .stack. Stack sets the top and bottom margin. This class gives breathing room for vertically stacked page elements. If .gutter is used on your grid columns this is done automatically.

.stack

.inline

Handle horizontal spacing with .inline. E.g. adding breathing room between an image and its paragraph on the right.

You can opt to use a .inline-group class on the parent element. This will effect its child elements to behave as if they are given the .inline style. Ideal for spacing elements in sequence like pills, tags, breadcrumbs or buttons.

.inset

Padding of an element can be managed by .inset, .stretched or .squished. Inset is the most common and sets a square padding (equal padding on four sides of the element).

.squished

A squished inset stretches horizontally. It reduces space top and bottom by 50% of the inset. This is mostly used for buttons and cell-like containers as tables or lists

.stretched

A stretched padding is the opposite of squished. It stretches vertically and is ideally for textboxes and other form elements

Responsive

Breakpoints

Since the develop mindset of warpaint is mobile first, the general idea is that you don't need to account for every viewport size in particular.

Having that said, there are some cases where working with breakpoints can be useful. Warpaint provides 5 breakpoints: 


/* extra small viewport (20em ~ 320px) and up */
@include xsmall-view{ ... }

/* small viewport (30em ~ 480px) and up */
@include small-view{ ... }

/* regular viewport (48em ~ 768px) and up */
@include medium-view{ ... }

/* large viewport (64em ~ 1024px) and up */
@include large-view{ ... }

/* extra large viewport (85em ~ 1365px) and up */
@include xlarge-view{ ... }

Grid

You can force the behaviour of columns and spacing classes in one of these breakpoints. Prepend the class with a breakpoint prefix like xs-, s-, m-, l- or xl-. E.g. Adding .l-column-3 to your column will enforce a 1/4 column size for viewports larger than 64em.

When using responsive column widths, make sure you also set the default .column-* / .column as well. This makes sure the column really behaves as a column.

.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6
Resize the viewport to view the change. Active breakpoint is .

Spacing

The same breakpoint prefixes can be applied to the spacing classes. E.g. .inset .l-inset-l will set standard inset but from the large breakpoint a large inset will be applied.

.inset .l-inset-l
